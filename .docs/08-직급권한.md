# 08 직급마스터-권한정책

## 1. 직급 마스터 분리
- `employee.position` 문자열을 `Position` 연관관계로 변경
- 직급 관리 기능 추가
  - `GET /positions`
  - `POST /positions/save`

## 2. 관리자/사원 권한 분리
- 역할: `ADMIN`, `EMPLOYEE`
- 로그인 계정
  - 관리자: `admin / 1234`
  - 사원: `employee / 1234`

## 3. 권한 정책
- 공통: 로그인 사용자만 시스템 접근 가능
- 관리자 전용
  - 사원 등록/수정/삭제
  - 부서 관리
  - 직급 관리
- 사원 권한
  - 사원 목록/상세/검색 조회만 가능

## 4. 구현 포인트
1. 인증/역할 모델
- `auth/UserRole`
- `auth/SessionUser`
- `auth/LoginUserModelAdvice`

2. 인터셉터
- `LoginCheckInterceptor`: 로그인 필수
- `AdminCheckInterceptor`: 관리자 전용 URL 차단

3. 서비스 계층 권한 검증
- `EmployeeService.save/update/delete`
- `DepartmentService.save`
- `PositionService.save`

4. 데이터 구조
- `positions` 테이블 추가
- `employees.position_id` FK 사용

## 5. UI 반영
- 헤더 메뉴에 직급 관리 추가
- 관리자에게만 등록/마스터관리 메뉴 노출
- 상태는 Enum 라벨(재직/휴직/퇴사)로 출력
