# 23-목록에러재발방지및일지복구

## 목표
- 게시판/사원목록 Mustache 키 에러 재발을 차단한다.
- 깨진 작업일지(.context) 문서를 복구한다.

## 원인 재정리
- 목록 템플릿에서 `condition.keyword`, `condition.employeeNo`를 직접 참조할 때
  모델 컨텍스트가 어긋나면 Mustache가 즉시 500을 발생시킴.
- `expose-request-attributes`가 켜진 상태에서는 컨텍스트 충돌 위험이 증가.

## 조치
1. 목록 템플릿 단순화
- `post/list.mustache`: `{{keyword}}` 사용
- `employee/list.mustache`: `{{employeeNo}}`, `{{name}}` 사용

2. 컨트롤러 모델 보강
- `PostController.list`: `keyword` 스칼라 전달
- `EmployeeController.list`: `employeeNo`, `name` 스칼라 전달

3. 설정 보강
- `application.properties`
  `spring.mustache.servlet.expose-request-attributes=false`

4. 작업일지 복구
- `.context/2026-02-09-12.md`
- `.context/2026-02-09-13.md`
- `.context/2026-02-09-14.md`
- `.context/2026-02-09-15.md`
- `.context/2026-02-09-16.md`
- `.context/2026-02-09-17.md`
- `.context/2026-02-09-19.md`

## 결과
- 목록 화면의 템플릿 키 충돌 리스크를 제거
- 깨진 작업일지 문서를 읽을 수 있는 상태로 복구
