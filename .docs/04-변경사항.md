# 04 변경사항 (Lombok, Layout, Test)

## 1. Lombok 정책 반영
- getter/setter/toString 수동 작성 대신 `@Data` 사용
- 적용 대상
  - `auth/LoginRequest`
  - `employee/Employee`
  - `employee/EmployeeRequest.SaveDTO`
  - `employee/EmployeeRequest.UpdateDTO`
  - `employee/EmployeeRequest.ListDTO`

## 2. Mustache 공통 레이아웃
- 공통 헤더: `templates/layout/header.mustache`
- 공통 푸터: `templates/layout/footer.mustache`
- 각 화면에서 partial include 방식으로 사용
  - `{{>layout/header}}`
  - `{{>layout/footer}}`

## 3. 더미데이터 확장
- `data.sql` 초기 데이터 2건 -> 12건으로 확대
- 목록/검색/페이징 확인이 쉽도록 부서/상태를 분산

## 4. 테스트 방식 변경
- MockMvc 기반 테스트 제거
- Service 직접 호출 방식으로 변경
  - `employee/EmployeeServiceIntegrationTest`
  - `auth/AuthServiceTest`

## 5. 이유
- 학습 난이도를 낮추고, 도메인 로직 중심 테스트를 우선 확인하기 위함
