# 25. 목록 키 에러 및 한글 메뉴 정리

## 배경
- 게시판(`/posts`) 접근 시 `No key ... 'keyword' on line 5` 예외로 500 발생
- 사원목록(`/employees`)도 동일 계열 키 누락 이슈 재발 가능성 존재
- 공통 메뉴가 HTML 엔티티 기반(`&#...`)으로 남아 있어 유지보수와 가독성이 떨어짐

## 원인
- Mustache 템플릿에서 점 표기 키(`condition.keyword`, `condition.employeeNo`)를 사용하면 현재 엔진에서 단일 문자열 키로 해석되어 키 조회 실패가 발생함
- 일부 화면은 컨트롤러에서 값을 전달하더라도 누락 시 바로 예외가 발생하는 구조였음

## 조치
- 검색 필드 키를 단일 키(`keyword`, `employeeNo`, `name`)로 통일
- `ViewDefaultModelAdvice` 추가로 기본값(`""`) 전역 주입
- `PostController`, `EmployeeController`에서 검색값을 명시적으로 모델에 주입
- `header/footer/home/post list/employee list` 문구를 실제 한글 텍스트로 교체
