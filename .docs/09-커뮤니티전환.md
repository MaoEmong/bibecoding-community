# 09 사내커뮤니티 전환

## 목표
- 기존 사원관리 중심 구조를 사내 커뮤니티 진입 흐름으로 전환
- 로그인 전: 로그인 페이지
- 로그인 후: 메인 페이지

## 핵심 정책
1. 회원가입 미제공
- 공개 회원가입 기능은 제공하지 않음

2. 관리자 사용자 추가
- 관리자가 `/admin/users`에서 사용자 데이터를 추가
- 아이디는 이메일 사용
- 비밀번호는 입력값(현재 더미 기준)

3. 역할 구분
- `ADMIN`, `EMPLOYEE`
- 관리자만 사용자/부서/직급/사원 등록 및 수정/삭제 가능

## 기본 계정
- 01 관리자: 이름 `홍길동`, 이메일 `ssar@nate.com`, 비밀번호 `1234`

## 추가 구현
- 사용자 도메인(`users`) 추가
- 루트 경로(`/`) 로그인 상태에 따라 로그인/메인 분기
- 메인 페이지(`/main`) 추가
- 관리자 사용자 관리 페이지(`/admin/users`) 추가
