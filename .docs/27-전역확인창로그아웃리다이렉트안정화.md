# 27. 전역 확인창/로그아웃/리다이렉트 안정화

## 배경
- 수정/삭제 작업에서 2차 확인 요구
- 로그아웃이 특정 화면에만 있어 접근성 낮음
- 일부 템플릿 한글 깨짐으로 화면 품질 저하
- 사용자관리 빈값 제출 시 안내 후 반복/멈춤 체감 발생

## 적용
1. 수정/삭제 확인창
- 게시글 수정, 사원 수정, 댓글 수정, 게시글 삭제, 사원 삭제, 댓글 삭제에 confirm 추가

2. 로그아웃 공통화
- 공통 헤더에 로그아웃 form 추가
- 사원목록 내 중복 로그아웃 영역 제거

3. 한글 깨짐 복구
- header/login/post save/post update/employee list/employee detail/comment update/user admin 템플릿 복구
- Summernote placeholder 문구 정상화

4. 리다이렉트 개선
- `UserAdminController`에서 입력 검증 실패를 직접 처리하여 `/admin/users`로 리다이렉트
- `GlobalExceptionHandler`에서 validation/internal 예외 시 referer 기반 redirectUrl 사용
