# 26. 게시글 삭제 확인 및 댓글 빈값 처리 개선

## 배경
- 게시글 삭제 버튼이 즉시 삭제되어 오동작 위험이 있음
- 댓글 내용 없이 제출 시 안내문구가 반복되는 체감 문제가 발생함

## 변경 내용
1. 게시글/댓글 삭제 확인
- `post/detail.mustache`의 삭제 form에 `confirm(...)` 추가

2. 댓글 빈값 제출 처리
- `CommentController`의 저장/수정 메서드에 `BindingResult` 추가
- 검증 실패 시 예외 페이지로 가지 않고 게시글 상세로 리다이렉트
- Flash Attribute(`commentError`)로 사용자 안내 메시지 출력

3. 상세 화면 문구 정리
- `post/detail.mustache`의 깨진 한글 문구를 실제 한글로 정리

## 변경 파일
- `src/main/resources/templates/post/detail.mustache`
- `src/main/java/com/example/bibecoding01/comment/CommentController.java`

## 검증
- `./gradlew.bat test` 통과

## 기대 효과
- 삭제 오클릭 방지
- 빈 댓글 제출 시 페이지 멈춤 체감 제거
- 사용자 피드백 경로(상세 화면 내 안내) 일관화
