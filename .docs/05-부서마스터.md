# 05 부서 마스터 분리

## 목표
- 사원 부서를 문자열 입력에서 마스터 데이터 참조 방식으로 전환
- 화면에서 부서 목록 선택으로 오타/미등록 부서 입력 방지

## 적용 내용
1. Department 도메인 추가
- `department/Department`
- `department/DepartmentRepository(Impl)`
- `department/DepartmentService`
- `department/DepartmentController`

2. Employee 구조 변경
- `Employee.department` : `String` -> `Department`(ManyToOne)
- 등록/수정 요청: `departmentId` 사용
- 목록 검색: `departmentId` 필터 사용

3. 화면 변경
- 사원 등록/수정: 부서 `select` 목록 사용
- 사원 목록 검색: 부서 `select` 필터 사용
- 상단 메뉴에 `Departments` 추가
- 부서 관리 화면 추가: `/departments`

4. 단순한 부서 추가 방식
- URL: `POST /departments/save`
- 입력: 부서명 1개
- 정책: 중복 이름 차단
- 이후 부서 추가는 화면에서 이름 입력으로 처리 가능

## 초기 기본 부서
- HR
- IT
- Finance
- Sales
- Operations
- Marketing
- Support
