# 24-게시판사원목록키에러후속조치

## 목표
- `keyword` / `employeeNo` Mustache 키 에러를 재발 없이 차단한다.

## 조치
1. 컨트롤러 모델 단순화
- `PostController.list`에서 `keyword`를 스칼라 모델로 전달
- `EmployeeController.list`에서 `employeeNo`, `name`을 스칼라 모델로 전달

2. 템플릿 참조 단순화
- `post/list.mustache`: `{{keyword}}`
- `employee/list.mustache`: `{{employeeNo}}`, `{{name}}`

3. 설정 보강
- `spring.mustache.servlet.expose-request-attributes=false`

4. 깨진 작업일지 복구
- `.context/2026-02-09-12.md`
- `.context/2026-02-09-13.md`
- `.context/2026-02-09-14.md`
- `.context/2026-02-09-15.md`
- `.context/2026-02-09-16.md`
- `.context/2026-02-09-17.md`
- `.context/2026-02-09-19.md`

## 검증 메모
- 코드/테스트는 통과.
- 런타임 검증은 로컬 환경의 Gradle wrapper lock(`.zip.lck`) 권한 충돌로 bootRun 기동이 차단되어 즉시 확인이 어려운 상태.
